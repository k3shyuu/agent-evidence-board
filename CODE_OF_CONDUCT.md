# Code of Conduct

This project follows a simple rule: be respectful, factual, and constructive.

We welcome issues and pull requests that improve evidence quality, safety, documentation, tests, or maintainability.

Unacceptable behavior includes harassment, spam, publishing secrets, doxxing, and using the project to support scams, credential theft, market manipulation, or platform abuse.

Maintainers may close issues or pull requests that violate these boundaries.
