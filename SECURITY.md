# Security Policy

## Reporting

Open a private security advisory if the repository host supports it, or contact the maintainers through the public project profile.

Do not publish exploit details or secrets in a public issue.

## Data boundaries

Agent Evidence Board is designed for public or explicitly approved data. Do not store:

- API keys;
- bot tokens;
- cookies;
- private keys;
- seed phrases;
- private repository content;
- customer data;
- private chat exports without permission.

The gate includes simple secret-pattern checks, but those checks are not a substitute for careful handling.

## Supported use

This project is intended for evidence tracking, review workflows, and publication gating. It is not intended for credential harvesting, spam, platform abuse, market manipulation, or automated trading.
